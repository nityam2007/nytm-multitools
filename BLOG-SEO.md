# Blog SEO Configuration | NYTM Multitools

## Overview
All blog URLs are **fully indexable** by Google Search Console with proper 301 permanent redirects to their corresponding tool pages.

## Architecture

### 1. Blog URL Structure
- Format: `/blog/{slug}` (e.g., `/blog/free-text-summary-online-tool`)
- Redirects to: `/tools/{toolSlug}` (e.g., `/tools/summary`)
- Redirect Type: **301 Permanent** (SEO-optimized)

### 2. Configuration Files

#### `/next.config.ts`
```typescript
async redirects() {
  return blogEntries.map((entry) => ({
    source: `/blog/${entry.blogSlug}`,
    destination: `/tools/${entry.toolSlug}`,
    permanent: true, // 301 status code
  }));
}
```
- Handles redirects at **edge/server level** (fastest)
- Returns **301 Permanent Redirect** status code
- Automatically syncs with blog entries from `lib/blog-info.ts`

#### `/app/sitemap.ts`
```typescript
const blogPages: MetadataRoute.Sitemap = blogEntries.map((entry) => ({
  url: `${BASE_URL}/blog/${entry.blogSlug}`,
  lastModified: currentDate,
  changeFrequency: "monthly" as const,
  priority: 0.6,
}));
```
- Includes all blog URLs in XML sitemap
- Generated dynamically from `blogEntries` array
- Accessible at: `https://nytm.in/sitemap.xml`

#### `/app/blog/[slug]/page.tsx`
- Generates **metadata** for each blog URL (title, description, OG tags)
- Sets `canonical` URL to actual tool page
- Sets `robots: { index: true, follow: true }` for full indexability
- Provides fallback UI in case redirect config fails

### 3. SEO Benefits

✅ **301 Permanent Redirects** - Google understands these are permanent URL consolidations  
✅ **Sitemap Inclusion** - All blog URLs submitted to GSC via sitemap.xml  
✅ **Proper Metadata** - Each blog URL has unique title/description  
✅ **Canonical URLs** - Points to actual tool page to avoid duplicate content  
✅ **Fast Redirects** - Handled at config level, not runtime  
✅ **No Redirect Chains** - Direct blog → tool mapping  

## Google Search Console Setup

1. **Submit Sitemap**: Add `https://nytm.in/sitemap.xml` in GSC
2. **Verify Indexing**: Check "Pages" report to see blog URLs indexed
3. **Monitor Redirects**: GSC will show 301 redirects in coverage report
4. **Check Canonicalization**: Verify canonical tags point to tool pages

## Blog Entry Management

Blog entries are auto-generated by:
```bash
node scripts/generate-blog-titles.js
```

This script creates SEO-optimized blog titles and updates `lib/blog-info.ts`.

## Verification Commands

```bash
# Test 301 redirect locally (after build)

# Expected response:
HTTP/1.1 301 Moved Permanently
Location: /tools/summary

# Check sitemap
curl https://nytm.in/sitemap.xml | grep "/blog/"
```

## File Structure
```
/app
  /blog
    /[slug]
      page.tsx          # Generates metadata + fallback UI
  sitemap.ts            # Includes blog URLs
/lib
  blog-info.ts          # Blog entries (auto-generated)
/next.config.ts         # 301 redirects configuration
/scripts
  generate-blog-titles.js  # Blog entry generator
```

## Notes
- All blog URLs return **301** status code (not 307 temporary)
- Metadata is generated even though users are redirected (for crawlers)
- Canonical URLs prevent duplicate content penalties
- Blog entries sync automatically with tools config
